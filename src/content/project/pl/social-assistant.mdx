Z powodu zmian w X API, aplikacja nie działa poprawnie. Została ona wyłączona.

## Pomysł

Większość aplikacji, stworzona do analizy liczby osób obserwujących profil na Twitterze, które przetestowałam działała zbyt wolno poprzez renderowanie po stronie serwera, albo interfejs aplikacji nie był zoptymalizowany na urządzenia mobilne.

Dlatego postanowiłam stworzyć nową aplikację przeznaczoną do tego celu.

## Stack technologiczny

Social assistant został stworzony przy użyciu stacku MERN:

- **M**ongoDB
- **E**xpressJS
- **R**eact
- **N**ode.js

### Backend

Strona backend Social assistant działa w środowisku Node.js.

#### Wykorzystane biblioteki:

- [Express](https://expressjs.com/) - framework Node.js
- [MongoDB](https://www.mongodb.com/) - baza danych
- [ws](https://github.com/websockets/ws) - serwer WebSocket używany do informowania o aktualnym stanie synchronizacji danych
- [passport.js](http://www.passportjs.org/) - wykorzystywany do autoryzacji konta Twitter
- [twitter-lite](https://github.com/draftbit/twitter-lite) - lekka biblioteka API Twittera
- [Agenda](https://github.com/agenda/agenda) - (Lekka biblioteka planowania zadań dla Node.js) używana do planowania aktualizacji danych użytkownika
